# WebSocket Client Project Knowledge

## Project Scope

1. Create a DSL for configuring WebSocket client state machines
2. Target XState v5 for state machine implementation
3. Use `ws` library for WebSocket functionality 
4. Focus on common WebSocket patterns

## Means of Achievement

1. State Machine: XState v5
2. WebSocket: `ws` library  
3. DSL: Maps to XState v5 configuration and ws usage

## Basic Requirements

### 1. Simplicity
- Clear, readable DSL syntax
- Standard WebSocket patterns only
- Minimal configuration needed
- Easy to understand behavior

### 2. Workability
- Handles common use cases
- Practical error handling
- Reasonable defaults
- Easy to debug

### 3. Stability 
- Small changes stay small
- Predictable behavior
- Consistent patterns
- Reliable operation

### Explicit Non-Goals
- Over-optimization
- Complex edge cases
- Unnecessary flexibility 
- Overly abstract solutions
