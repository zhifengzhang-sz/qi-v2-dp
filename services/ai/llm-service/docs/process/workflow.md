# Implementation Workflow

## Problem Analysis

- **Design Check**: Review `/docs/impl/*.md` first
- **Component Impact**: Map to design components
- **Requirements**: Verify against original needs

## Solution Planning

1. Document the issue
2. Review design implications
3. Plan component changes
4. Write tests
5. Implement solution

## Documentation

- Update relevant design docs
- Record decisions made
- Note any deviations

## Implementation Rules

1. No code changes without design review
2. Follow component boundaries
3. Test first, then implement
4. Document all decisions
